# AI-assisted formalization workflow

AI-generated theorem statements and proof attempts are useful only after verification. The workflow below treats each output as a proposal, not as trusted content.

| Agent role | Input | Output | Verification |
|---|---|---|---|
| literature parser | Morgan--Tian/Kleiner--Lott | theorem/source map | domain expert review |
| theorem statement agent | source theorem + mathlib API | Lean statement sketch | `lake env lean` + reviewer |
| mathlib search agent | informal goal | LeanSearch/Moogle/docs report | citations + `#check` |
| skeleton agent | statement table | `.lean` files with pending theorem | CI compile |
| proof attempt agent | leaf lemma | proof attempts | local no-sorry test |
| adversarial audit agent | blueprint node | missing assumptions report | human approval |
| CI repair agent | build logs | minimal edits | CI rerun |
| expert review loop | PR + source map | approval or corrections | reviewer sign-off |

## Anti-hallucination rules

1. A theorem name generated by an LLM is `unverified` until `#check` succeeds.
2. Every mathematical statement needs a source locator.
3. Every proof sketch must name dependencies.
4. No Ricci/surgery/extinction proof may collapse into “standard argument”.
5. Every PR needs Lean review and mathematics review.
6. A weekly report must list pending theorems and stale blueprint nodes.
